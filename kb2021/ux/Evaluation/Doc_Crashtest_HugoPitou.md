# Crash test

## Crash test = Test utilisateur

Le Crash test est plus communément appelé Test Utilisateur ou Test Ux

---

## Objectif du test utilisateur ?

-> il permet de tester le bon fonctionnement du site

-> il permet de valider les fonctionnalités et d'identifier les fonctionnalités manquantes

->Les tests utilisateurs permettent justement de se prémunir de ce genre de mauvaises surprises

-> il peut servir à effectuer des tests utilisateurs permettant de tester l’ergonomie du site

---

## Comment l’organiser

-> La personne qui sera le testeur ne doit pas connaitre le logiciel a l'avance pour ne pas lui facilité le test utilisateur pour avoir des ressenti de la personne commme d'un nouvelle utilisateur.

-> Laisser a l'utilisateur quelques minutes pour regarder le logiciel pour qui le découvre.

-> Ensuite lui expliquer le scenario qui devra suivre sans lui donner de conseil sur le logiciel.

-> Puit faire un debriefing de son experience utilisateur et voir les amélioration et les chose qui vont bien.

---

## Quand faire des tests utilisateurs

il n'y a pas de moments bien précis pour faire un test utilisateur selon les entreprise et les groupe de dev le nombre de test utilisateur varie.
